# Definition of Done (DoD)

Фича считается **готовой к production** только если выполнены ВСЕ пункты:

---

## ✅ Код

- [ ] Код написан и работает по happy-path
- [ ] Обработаны edge cases и ошибки
- [ ] Нет жёстко зашитых строк — все тексты в `locales/`
- [ ] Нет секретов в коде — всё через `.env`
- [ ] Пользовательский ввод экранируется перед вставкой в сообщения

## ✅ Тесты

- [ ] Написаны unit-тесты для ключевой логики
- [ ] Покрытие новых функций ≥ 70%
- [ ] Все тесты проходят (`pytest`)
- [ ] Линтеры не ругаются (`black`, `isort`, `flake8`)

## ✅ Безопасность

- [ ] Запущен `pip-audit` — критических уязвимостей нет
- [ ] Нет прямых пушей в `main` — только PR
- [ ] Права доступа проверены (нельзя получить чужие данные)
- [ ] Rate limiting работает для новой точки входа

## ✅ База данных

- [ ] Написана миграция Alembic (если изменялась схема)
- [ ] Миграция проверена на откат (`alembic downgrade -1`)
- [ ] Новые запросы используют параметризацию (без f-string в SQL)

## ✅ Документация

- [ ] Обновлён README (если изменился запуск/конфиг)
- [ ] Обновлены комментарии в сложных местах кода
- [ ] Если принято архитектурное решение — создан ADR

## ✅ Мониторинг

- [ ] Критические ошибки логируются в Sentry
- [ ] Добавлены метрики/события в аналитику (если нужно)

---

> ⚠️ Без выполнения всех пунктов задача остаётся в статусе **"In Review"** и не переходит в **"Done"**.
