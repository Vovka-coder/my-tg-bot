# Политика работы с секретами

## Правила (обязательны для всех участников)

1. **Никогда не коммитить** `.env`, токены, пароли, приватные ключи.
   - `.gitignore` защищает от случайного коммита
   - GitHub Secret Scanning уведомит при обнаружении

2. **Передача секретов** только через:
   - Зашифрованные сообщения (Signal, Telegram Secret Chat)
   - Менеджер паролей (Bitwarden, 1Password)
   - При появлении второго разработчика — Doppler или HashiCorp Vault

3. **Хранение локально** — только в `.env` файле, никогда не в коде или комментариях.

4. **Ротация:**
   - Плановая: каждые 3–6 месяцев
   - Экстренная: немедленно при подозрении на утечку

## Экстренная процедура при утечке токена бота

```bash
# 1. Отозвать токен
# Открыть @BotFather → /mybots → Выбрать бота → API Token → Revoke

# 2. Заменить в .env
BOT_TOKEN=новый_токен

# 3. Перезапустить бота
docker-compose restart bot

# 4. Проверить логи на признаки злоупотребления
docker-compose logs bot --since 24h | grep -i "unauthorized\|forbidden"

# 5. Задокументировать инцидент в docs/incidents/
```

## Переменные по уровню критичности

| Критичность | Переменные | Ротация |
|-------------|-----------|---------|
| 🔴 Высокая | BOT_TOKEN, DB-пароли, STRIPE_SECRET_KEY | Немедленно при утечке |
| 🟠 Средняя | OPENAI_API_KEY, SENDGRID_API_KEY | 3 месяца |
| 🟡 Низкая | S3_ACCESS_KEY, SENTRY_DSN | 6 месяцев |

## GitHub Actions Secrets

Все секреты для CI/CD добавлять в **Settings → Secrets and variables → Actions**,
никогда не прописывать их в `.yml` файлах напрямую.
